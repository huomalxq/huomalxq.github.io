# AI 辅助整理学习记录

## 背景

学习过程里经常会有命令、判断、失败尝试和临时想法。如果只在脑子里过一遍，很容易最后只剩一个结论，忘掉当时为什么这么做。AI 工具适合参与“整理”和“复盘”环节，把零散材料转成结构清晰的学习记录。

## 过程

比较适合的协作方式是先给 AI 三类信息：

- 这次学习要解决的问题。
- 已经尝试过的命令、配置或资料。
- 最终希望沉淀成什么形式，例如站点笔记、复盘文档或待办清单。

整理时可以沿用本站的笔记模板：

```md
## 背景

## 过程

## 结论

## 后续
```

其中“过程”不需要写成流水账，而是保留对以后有价值的信息：关键命令、错误提示、判断依据、替代方案和最终选择。

## 结论

AI 更适合帮助学习记录变得完整，而不是替代学习本身。一个比较好的分工是：

- 人负责提供真实上下文和最终判断。
- AI 负责整理结构、补全表达、提醒遗漏点。
- 最后仍由人复查事实、命令和结论是否准确。

## 后续

后续可以继续补充一个专门的 Prompt 模板，用于把一次学习过程快速整理成本站格式的 Markdown 页面。
