# 工程实践

这一栏记录偏工程落地的内容：环境搭建、依赖管理、部署发布、调试排障、自动化脚本和文档化流程。目标是把一次性解决的问题整理成下次可以直接复用的清单。

## 当前内容

- [Git 仓库维护流程](git-repository-workflow.md): 记录新增学习笔记时的 Git 与构建检查流程。
- [MkDocs 导航维护](mkdocs-navigation.md): 记录新增页面后如何同步维护站点导航。
- [学习记录模板](note-template.md): 适合作为后续每篇技术笔记的起始格式。

## 记录范围

### 环境与依赖

- Python / Node / Docker 等运行环境的安装、隔离和版本记录。
- 依赖冲突、镜像源、权限问题和缓存目录处理。
- 可复现环境所需的最小命令和配置文件。

### 构建与部署

- MkDocs、GitHub Pages、GitHub Actions 等静态站点发布流程。
- Nginx、Docker、FastAPI / Flask 服务部署链路。
- 构建失败、路径错误、静态资源缺失等问题的排查路径。

### 调试与自动化

- 用脚本统计实验指标、接口耗时、日志异常和端到端响应时间。
- 记录“如何定位问题”，而不仅是“最后怎么修好”。
- 把重复操作整理成命令片段、检查清单或小工具。

## 推荐记录格式

- 背景：问题来自哪里，影响什么流程。
- 现象：报错、日志、异常输出或指标变化。
- 排查：尝试过哪些方向，排除了什么可能。
- 解决：最终修改的命令、配置或代码。
- 复用：下次遇到同类问题时可以直接照做什么。
