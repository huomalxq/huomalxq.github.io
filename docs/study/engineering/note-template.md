## 使用建议

- 标题写清“对象 + 动作 + 场景”，例如“MkDocs 静态资源路径检查”。
- 过程里保留失败尝试，因为失败尝试通常比最终命令更能体现判断。
- 如果是实验笔记，优先保留指标口径、对比组和结论适用范围。
- 如果是部署 / 排障笔记，优先保留环境、日志、最小复现和复用清单。
